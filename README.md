# Countdown to August 5th, 2026

A personal countdown site — two people, one day.

Every day a new AI-generated Bob Ross-style painting is commissioned, animated into a living video, and used as the full-screen background. The countdown clock ticks down to midnight in the viewer's own timezone. A growing gallery archives every painting along with the real weather, live visitor data, and uploaded personal memories.

Running 24/7 on a homelab server behind Cloudflare Tunnel.

---

## How it works

Each day at midnight, the app runs an automated pipeline:

**1. Find a holiday**
A public holidays API is queried to find every celebration happening somewhere in the world that day. The most visually interesting one is selected.

**2. Check the weather**
The current conditions in Lehi, Utah are fetched from [Open-Meteo](https://open-meteo.com/) — temperature, precipitation, wind, and WMO weather code. This is translated into a weather effect type (sun, rain, snow, storm, fog, wind, heat, aurora) and an intensity value from 0 to 1.

**3. Claude writes the prompt**
Claude Sonnet 4.6 receives the holiday context and weather data. It writes a 250–350 word image generation prompt describing a complete Bob Ross oil painting scene — sky, mountains, trees, water, a culturally specific focal element, real Bob Ross paint colors, and lighting conditions.

**4. Flux 2 Pro generates the image**
The prompt is sent to [Replicate](https://replicate.com)'s API where `black-forest-labs/flux-2-pro` renders a 1008×1008 PNG. If the image scores below 7.5/10, a revised attempt is made. Up to 3 attempts; the best result wins.

**5. Claude scores the result**
The finished image is evaluated by Claude Sonnet 4.6 using vision input — scored on painterly quality, cultural accuracy, composition, and color. The score, critique, dominant colors, and a short theme description are extracted.

**6. Stable Video Diffusion animates it**
The winning PNG is uploaded to Replicate file hosting and passed to `stability-ai/stable-video-diffusion`. SVD generates 25 frames of subtle motion — clouds moving, water rippling, light shifting — encoded as a looping MP4. The static PNG is replaced by the living video as soon as it's ready.

**7. Everything is cached on disk**
PNG, MP4, weather JSON, and art metadata are written to the homelab server. Nothing is regenerated if it already exists.

**8. Viewers are tracked in SQLite**
Every page load geolocates the visitor via ip-api.com (country + US state) and logs it to SQLite. These are shown live in the sidebar and permanently archived with each gallery entry.

## Features

- **Animated AI painting** — Bob Ross-style oil painting generated by Flux 2 Pro, animated into a looping MP4 by Stable Video Diffusion. A brand new scene every single day.
- **Real weather integration** — Open-Meteo weather in Lehi, UT is baked into the painting prompt and reflected as a particle overlay (rain drops, snow, aurora, etc.)
- **Timezone-aware countdown** — the clock counts down to midnight August 5th in *your* local timezone, not the server's
- **Milestone celebrations** — full-screen confetti effects at 365, 100, 50, 25, 10, 5, 3, 2, and 1 days remaining
- **Growing gallery** — every past painting is archived with its holiday, score, critique, viewer count, and weather
- **Timelapse player** — watch all paintings play back in sequence inside the gallery
- **Live visitor sidebar** — real-time display of who's viewing right now, with country and US state flags
- **Personal memories** — upload photos to any gallery day alongside the AI painting
- **Score-based refinement** — Claude evaluates each image and up to 3 attempts are made to exceed a 7.5/10 threshold

## Infrastructure

The app runs inside a Debian LXC container on a Proxmox homelab server. A Cloudflare Tunnel (cloudflared) runs inside the container and creates an outbound-only encrypted connection to Cloudflare's edge — no router port forwarding required. Public traffic hits Cloudflare, routes through the tunnel, and is served with full TLS.

Every push to `master` triggers a GitHub Actions workflow that SSHes into the server, runs `dotnet publish`, copies the output into the container, and restarts the systemd service. API keys are stored as GitHub Secrets and written to `/etc/homecountdown.env` at deploy time.

Art files survive deploys — PNG, MP4, JSON, and SQLite are stored at `/var/lib/homecountdown/art-cache/` and mounted into the container independently of the app binary.
